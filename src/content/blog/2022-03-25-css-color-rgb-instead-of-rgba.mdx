If you want to support transparency in a CSS `rgb` or `hsl` function, there's no need to write `rgba` or `hsla` anymore. You can simply write `rgb` or `hsl` with a `/` to indicate the alpha.

No need for commas too!

{/* more */}

Here's an example.

```css
.rgb {
  rgb(59 50 103 / 0.3)
}

.hsl {
  color: hsl(250 35% 30% / 0.3);
}
```

Here's the good news: **This is supported across all major browsers** â€” so you can use it in production today!

You can also use this if you're a Sass user â€” **Dart Sass supports this syntax too**, but it converts the function into a regular `rgba` underneath the hood.

```scss
// What you write
.selector {
  color: hsl(250 35% 30% / 0.3);
}

// Compiled CSS
.selector {
  color: rgba(59, 50, 103, 0.3);
}
```

That's it!
